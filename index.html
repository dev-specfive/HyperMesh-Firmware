🛰️ What is HyperMesh

HyperMesh is Spec5's LoRa mesh communication firmware and app ecosystem for off-grid messaging and field connectivity.

It runs on supported ESP32-S3 + LoRa development boards and is compatible with Spec5 hardware as well as selected third-party hardware platforms.

🚀 Flashing Instructions (Board-Based)

Prerequisites

Before flashing, ensure you have:

  • One supported development board
  • USB cable for connecting the board to your computer
  • Appropriate USB drivers installed (for example, CP210x or CH340)
  • The correct board-specific firmware file from the board-based download table above

Required Tools

You will need the ESP32 Flash Download Tool to flash firmware to ESP32-S3 based boards.

🔗 Download ESP32 Flash Download Tool

Enter Download Mode

Download-mode behavior can vary by board design:

  • Heltec Tracker V1.2: Hold USER/BOOT, tap RST, then release USER/BOOT.
  • Heltec Wireless Stick V3.1 / LoRa32 V3: Often enters download mode automatically when flashing starts; if not, use the BOOT+RST method above.

Step-by-Step Flashing Instructions

Reference: ESP32-S3 Flash Download Tool V3.9.9_R2 - Successful flash completion

  1. Connect Board
    Connect the supported board to your computer via USB.
  2. Open Flash Tool
    Launch the ESP32-S3 Flash Download Tool on your computer. Select ESP32-S3 as the hardware type.
  3. Select COM Port
    Choose the COM port (Windows) or Serial port (macOS/Linux) where your board is connected.
  4. Configure Flash Settings
    Set the following parameters:
    • Flash Address: 0x000
    • Hardware: ESP32-S3
    • SPI Speed: 40MHz (recommended)
    • SPI Mode: DIO (default)
  5. Load Board Firmware
    Select the correct board-specific .bin file from the firmware download section.
  6. Start Flashing
    Click START to begin uploading the firmware to your device. Wait for the flashing process to complete (progress bar will show status). You should see a "FINISH" message (完成) when successful, as shown in the reference image above.
  7. Reboot
    Reboot the board/device and validate connectivity in the HyperMesh app.

📝 Notes

  • Product name = what the user bought.
  • Board name = underlying hardware platform.
  • Firmware file = what gets flashed to the board.
  • Always flash firmware based on board compatibility, not product marketing name.

🔧 Troubleshooting

Device Not Detected

  • Ensure the correct USB drivers for your device are installed
  • Try a different USB cable (data-capable cable required)
  • Try a different USB port on your computer
  • Check Device Manager (Windows) or System Information (macOS) for COM port detection

Flashing Fails to Start

  • Put the board in download mode (BOOT+RST sequence).
  • Confirm board hardware is set to ESP32-S3.
  • Confirm flash address is set to 0x000.
  • Verify selected firmware matches the board type.
  • Check that the correct COM port is selected

Flashing Process Interrupted

  • Do not disconnect the device during flashing
  • Ensure stable USB connection
  • Close other applications that might be using the serial port
  • Retry the flashing process

Device Not Responding After Flash

  • Reflash the board using the correct board firmware.
  • Lower baud rate and retry if flashing was unstable.
  • Power cycle the device after a successful finish state.
